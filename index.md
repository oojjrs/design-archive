# 모눈세계 (Core Design Document)

## 1. 프로젝트 정의

이 프로젝트는 단순한 시뮬레이션 게임이 아니다.

> **"세계들을 생성하고, 축적하고, 충돌시키며, 감정을 만들어내는 시스템"**

플레이어는 세계를 플레이하는 것이 아니라:

* 세계를 만든다
* 세계를 관찰한다
* 세계에 개입한다
* 세계의 결과를 다음 세계로 이어간다

---

## 2. 핵심 루프

```
창조 → 관찰 → 개입 → 결과 → 추출 → 재투입
```

### 설명

* **창조**: 지형, 환경, 생명체(DNA)를 설정
* **관찰**: 개입 없이 세계의 변화를 지켜봄
* **개입**: 제한된 자원(신앙심)을 사용하여 영향 행사
* **결과**: 생태계 변화, 문명 형성 또는 붕괴
* **추출**: 의미 있는 결과를 Artifact로 저장
* **재투입**: 다음 세션 또는 다른 세계에 영향

---

## 3. 핵심 리소스

### 1) 시간

* 자동으로 축적됨
* 세계의 가치 자체

### 2) 신앙심

* 플레이어의 개입 리소스
* 강력하지만 제한적

---

## 4. Artifact 시스템

세션의 결과물은 다음 세계에 영향을 주는 단위로 변환된다.

### 종류

* **Species (종)**
* **Civilization (문명)**
* **Technology (기술)**
* **Event (사건)**
* **Relic (유물)**

### 역할

> Artifact는 단순 데이터가 아니라, 다음 세계의 조건이 된다.

---

## 5. 세션 간 연결 구조

### 1) 계승 (Inheritance)

* 자신의 세계가 다음 세계로 이어짐

### 2) 충돌 (Conflict)

* 다른 세계와 경쟁

### 3) 융합 (Fusion)

* 서로 다른 세계의 요소가 결합

---

## 6. 세계의 레이어 구조

### Layer 1 — 환경

* 지형, 기후, 자원

### Layer 2 — 생명

* DNA 기반 개체
* 번식, 진화

### Layer 3 — 문명

* 사회, 문화, 기술

### Layer 4 — 초월

* 신 개입
* 신앙심 시스템

### Layer 5 — 외부 영향

* 다른 유저
* 외계 요소

> 각 레이어는 독립적으로 동작해야 한다.

---

## 7. DNA 설계 원칙

DNA는 수치가 아니라 "행동과 형태"를 만들어야 한다.

```
DNA → 행동 변화 → 형태 변화 → 생태계 결과
```

### 최소 구성 예시

* Size
* Speed
* Aggression
* Fertility
* Perception

### 시각 매핑

| DNA        | 표현        |
| ---------- | --------- |
| Size       | 몸 크기      |
| Speed      | 다리 길이     |
| Aggression | 턱 / 공격 기관 |
| Fertility  | 복부        |
| Perception | 눈         |

---

## 8. 아트 방향성

아트는 "미적 요소"가 아니라 **정보 전달 시스템**이다.

### 원칙

1. 변화는 눈에 띄어야 한다
2. 변화는 의미를 가져야 한다
3. 변화는 계승되어야 한다

### 접근 방식

* 모듈 조합형 (초기)
* 파라메트릭 변형 (확장)
* 상징적 표현 (전략 단계)

---

## 9. 감정 루프

```
창조 → 애착 → 확장 → 위협 → 붕괴 → 회고 → 재설계
```

### 핵심 감정

* 보호
* 통제
* 상실
* 초월

---

## 10. 신과 세계의 관계

### 단계

1. 의존
2. 자율
3. 저항
4. 독립
5. 초월

> 목표: 세계가 플레이어의 개입 없이도 유지되고, 결국 개입을 거부한다.

---

## 11. 멀티플레이 구조

### 비동기 (핵심)

* 다른 유저의 Artifact 유입
* 교배, 침입, 거래

### 동기 (보조)

* 세션 기반 경쟁
* 생태계 대결

---

## 12. 멀티버스 구조

### 분기 방식

* 선택 기반 분기
* 이벤트 기반 자연 분기

### 특징

* 세계는 가지처럼 갈라진다
* 일부는 다시 만나거나 영향을 준다

---

## 13. 세계의 생명성

생명체가 없어도 세계는 살아 있어야 한다.

### 조건

* 지속적인 변화
* 환경 반응성
* 시간 기반 패턴

---

## 14. 핵심 재미 요소

1. 내가 만든 것이 다시 등장한다
2. 예상 못한 조합이 발생한다
3. 실패가 다음 세계에 영향을 준다
4. 세계의 역사가 축적된다

---

## 15. MVP 목표

> **"내가 만든 생명체가, 내 개입 없이도 살아남는다"**

### 최소 구현

* 단일 종
* 번식 / 죽음
* 환경 영향
* 간단한 개입 시스템

---

## 16. 기술 설계 원칙

* 데이터 중심 구조
* deterministic simulation (seed 기반)
* 이벤트 로그 기반 기록
* ID 기반 관계 (직접 참조 금지)

---

## 17. 최종 목표

> **"세계가 태어나고, 살아가고, 충돌하고, 사라지고, 다시 이어지는 시스템"**

플레이어는:

* 창조자
* 관찰자
* 개입자
* 기록자

이며,

결국

> **세계의 일부가 된다**
